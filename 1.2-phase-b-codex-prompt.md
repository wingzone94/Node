## タスク1: F-5 — 範囲外ページネーション301モジュール

### 意図
「一度生きていたページネーションURL」が posts_per_page 変更で404になったものを、同アーカイブの最終有効ページへ301リダイレクトする。

### 実装仕様
1. `functions.php` 末尾に独立関数群を追加（将来の `inc/routing.php` 集約に備え、関数は自己完結させる）。
2. **純関数** `node_resolve_404_redirect_target( WP_Query $wp_query, string $request_uri ): ?string` を作成。この関数はリダイレクト先URL文字列を返すか、nullを返す。exitしない。HTTP操作しない。テスト可能にするため。
3. **薄いフック関数** `node_handle_404_redirect()` を `template_redirect` priority 1（`node_spotlight_404_fallback` の priority 0 の後）に登録。`is_404()` でなければ即return。404のときのみ純関数を呼び、非nullなら `wp_safe_redirect($url, 301); exit;` する。
4. 純関数の判定ロジック:
   - **(a) paged付きアーカイブ（カテゴリ/タグ/日付/シリーズ/検索）**: `$wp_query->is_paged()` が true かつ `$wp_query->get('paged')` > 0 の場合。ただし is_404 状態では WP_Query のフラグが信頼できないため、`$request_uri` から `/page/(\d+)/` をパースして検出する。
     - `$request_uri` から `/page/\d+/` 部分を除去したベースURLを構成
     - そのベースURLに対する WP_Query を `new WP_Query()` で実行し `$found_posts` と `posts_per_page` から `$max_page` を算出
     - 要求ページ > `$max_page` なら、`$max_page` == 1 の場合はベースURL（/page/なし）へ、`$max_page` >= 2 の場合は `{base}/page/{$max_page}/` へリダイレクト
     - クエリ文字列（`$_SERVER['QUERY_STRING']`）があれば維持
   - **(b) 分割記事の範囲外ページ**: `$request_uri` が `/{post-slug}/\d+/` パターンで、該当投稿が存在し `$post->post_content` 内の `<!--nextpage-->` 数+1 < 要求ページ番号の場合 → 記事1ページ目（`get_permalink($post)`）へ301
   - **(c) `/all-articles/page/N/`**: `template-parts/all-articles.php` が独自に404処理しているが、そこに到達する前に本モジュールが301で救済する。`NODE_ALL_ARTICLES_TOTAL_LIMIT`（定数）と `NODE_ALL_ARTICLES_PER_PAGE` から `$max_page` を算出。
   - **(d) それ以外**: null を返す（404維持）。
5. **注意**: `node_spotlight_404_fallback`（priority 0）は `/spotlight/` を200に書き換える。priority 1で発火する本フックは `is_404()` チェック済みなので干渉しない。`node_redirect_spotlight_category_archive`（priority 10、`/category/spotlight/` → `/spotlight/` の301）とも独立。

### 対応テスト: T-2（`tests/node-pagination-redirect-test.php` 新規）

STRUCTURAL-REVIEW-1.2.md の T-2 節に準拠。以下のケースを網羅:

```
301になるべきケース:
1. /category/{slug}/page/99/ → /category/{slug}/page/{max}/
2. /category/{slug}/page/{max+1}/ → /category/{slug}/page/{max}/
3. /page/99/ (home) → /page/{max}/
4. /all-articles/page/99/ → /all-articles/page/{max}/
5. 分割記事 /post-slug/9/ (numpages=3) → /post-slug/
6. ?utm_source=x 付き → クエリ文字列維持で301

404のまま維持（リダイレクトしない）:
7. /no-such-page/
8. /category/no-such/page/2/（存在しないterm）
9. /series/no-such/

範囲内（リダイレクト発生ゼロ＝最重要回帰）:
10. /category/{slug}/page/{max}/ → リダイレクトなし
11. /post-slug/2/ (numpages=3) → リダイレクトなし
```

純関数を直接テスト（`$wp_query` と `$request_uri` を手動構成して渡す）。WP_UnitTestCase を使い、`factory->post->create_many()` でシードデータを作成。

## タスク2: F-6 — 404.php 新設

### 意図
404ページにnoindex・検索フォーム・主要導線を持つ軽量テンプレートを設置し、404の増幅器を解消する。

### 実装仕様
1. **`404.php`** を新規作成。`index.php` の構造（`get_header()`/`get_footer()`・`m3-*` クラス）を踏襲:
   ```php
   <?php get_header(); ?>
   <main id="primary" class="site-main m3-home-layout">
       <?php node_the_breadcrumbs(); ?>
       <section class="m3-404-section m3-surface m3-section-spacing">
           <h1 class="m3-404-title">ページが見つかりません</h1>
           <p class="m3-404-message">お探しのページは移動または削除された可能性があります。</p>
           <div class="m3-404-search">
               <?php get_search_form(); ?>
           </div>
           <nav class="m3-404-nav" aria-label="主要ページ">
               <ul>
                   <li><a href="<?php echo esc_url( home_url( '/' ) ); ?>">ホーム</a></li>
                   <li><a href="<?php echo esc_url( node_get_all_articles_url() ); ?>">全記事一覧</a></li>
                   <li><a href="<?php echo esc_url( home_url( '/headlines/' ) ); ?>">ヘッドライン</a></li>
               </ul>
           </nav>
       </section>
   </main>
   <?php get_footer(); ?>
   ```
2. **`wp_robots` フィルタで404時 noindex を追加**: `functions.php` に以下を追加:
   ```php
   function node_robots_noindex_404( array $robots ): array {
       if ( is_404() ) {
           $robots['noindex'] = true;
           $robots['nofollow'] = true;
       }
       return $robots;
   }
   add_filter( 'wp_robots', 'node_robots_noindex_404' );
   ```
3. **`src/styles/_404-lite.css`** を新規作成。最小限のスタイル（タイトル・メッセージ・検索フォーム・ナビリンクのレイアウト）。Material Design 3のトーンに合わせる。
4. **`src/styles/style.css`** に `@import './_404-lite.css';` を追加（`_layout-fixes.css` の後あたり）。
5. デッドの `src/styles/_404.css`（559行）は触らない。

### 対応テスト: T-4（`tests/node-404-template-test.php` 新規）

```
1. go_to('/definitely-404-xyz/') → is_404() true
2. 404テンプレートが 404.php を返す（get_404_template()）
3. wp_robots フィルタに noindex が含まれる（404時）
4. wp_robots フィルタに noindex が含まれない（通常ページ、回帰）
```

## タスク3: F-7 — Series移行301（旧URL→/series/）

### 意図
連載を旧来表現していたカテゴリ/タグURLから新 `/series/{slug}/` への301リダイレクトで、SEO資産を移送する。

### 実装仕様
1. `plugins-embedded/node-series/node-series.php` の `Node_Series` クラスに以下を追加:
2. **純関数（staticメソッド）** `resolve_legacy_redirect( ?object $queried_object, string $request_uri ): ?string`
   - option `node_series_redirect_map`（連想配列、初期値は空）を `get_option()` で取得
   - マップ形式: `{'category:old-slug' => 'series-slug', 'post_tag:old-tag' => 'series-slug'}`
   - `$queried_object` が WP_Term で、`"{$queried_object->taxonomy}:{$queried_object->slug}"` がマップにあれば:
     - マップ先の series term が存在するか `get_term_by('slug', $target, 'node_series')` で確認
     - 存在しなければ null（リダイレクトしない＝安全弁）
     - 存在すれば `/series/{slug}/` を構成。`/feed/` が `$request_uri` に含まれていれば `/series/{slug}/feed/` に。ページ位置は引き継がない。
     - クエリ文字列は維持
   - マップ未登録 / `$queried_object` が null / taxonomy が category/post_tag 以外 → null
3. **薄いフック** `handle_legacy_redirect()` を `template_redirect` priority 5 で登録（`node_redirect_spotlight_category_archive` の priority 10 より前。ただし spotlight の `/category/spotlight/` は独自301があるため、マップに spotlight を入れなければ干渉しない）:
   - `is_admin()` なら return
   - `is_category() || is_tag()` でなければ return
   - `get_queried_object()` を渡して純関数を呼び、非nullなら `wp_safe_redirect($url, 301); exit;`
4. マップの初期値は**空配列**。投入はWP-CLI/コードで行う（管理画面UIは作らない）。

### 対応テスト: T-3（`tests/node-series-redirect-test.php` 新規）

STRUCTURAL-REVIEW-1.2.md の T-3 節に準拠:

```
前提: マップ {'category:yugioh-old' => 'yugioh', 'post_tag:renai-tag' => 'renai'} をシード。
series term 'yugioh' と 'renai' を作成。カテゴリ 'yugioh-old' とタグ 'renai-tag' を作成。

301すべき:
1. /category/yugioh-old/ → /series/yugioh/
2. /category/yugioh-old/page/2/ → /series/yugioh/（ページ引継ぎなし）
3. /category/yugioh-old/feed/ → /series/yugioh/feed/
4. /tag/renai-tag/ → /series/renai/
5. ?utm=x 付き → クエリ維持

301してはならない:
6. マップ未登録カテゴリ → null
7. マップ先 series term が削除済み → null
8. is_admin 時 → 発火しない
```

純関数 `resolve_legacy_redirect()` を直接テスト（`$queried_object` を手動構成して渡す）。
