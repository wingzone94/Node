# 前提
- 対象ファイル: inc/blogcard.php（正本）、plugins-embedded/luminous-nexus/includes/shortcode-blogcard.php（プラグイン版、重複実装あり）、src/styles/_blogcard-ai.css

- 発端の不具合: core/embedブロックで自サイトの記事URLが正しくカード化されない（他のWordPressサイトの記事URLは正常）。原因は node_auto_blogcard() が the_content 優先度11の正規表現スキャンであるため、優先度8のWP_Embed::autoembed()に先を越されること。
- 方針転換: 正規表現スキャンをやめ、WP標準のoEmbed解決パイプライン（oembed_dataparse / embed_maybe_make_link フィルター）にフックしてカード化する。

- **X(Twitter)・YouTubeは対象外（重要）**。ユーザー明示指示により、これらのURLはWP標準のネイティブ埋め込みコードを一切変更しない。oembed_dataparseフック内で最初に除外判定を行うこと。
- AGENTS.md の制約に従うこと（1タスク1目的、原則5ファイルまで、全体リライト禁止、bun x vite build必須）。

# やること（設計判断のみ、実装はしない）
1. oembed_dataparse フックの関数シグネチャと、X/YouTube除外判定の具体的な実装方法（$data->provider_nameの文字列マッチか、$urlのドメインマッチか、両方か）を決める。対象ドメイン: twitter.com, x.com, youtube.com, youtu.be。
2. 除外対象でない場合の、$data（title / thumbnail_url / provider_name / author_name）からm3-blogcard HTMLへの変換ロジックを、既存のnode_render_blogcard()のHTML構造（m3-blogcard__content / __text / __title / __footer / __image など）に揃える形で具体化する。
3. embed_maybe_make_link フックでの既存node_render_blogcard()への委譲方法を決める。
4. node_auto_blogcard()（the_content優先度11）の廃止方法（フィルタ登録解除のみか関数削除まで含むか）を決める。
5. 既存oEmbedキャッシュ（投稿メタ _oembed_<hash> / _oembed_<hash>_time）の無効化・検証手順を、Phase 3で使える具体的なコマンド/手順として決める。
6. PHPUnitテストでカバーすべき項目をリストアップする（oembed_dataparseのX/YouTube除外分岐、データ→HTML変換、embed_maybe_make_linkの委譲、既存ショートコード・OGPフォールバックの回帰）。

# 出力
上記の決定事項を、GPT-5.5がそのままPhase 2aの実装に着手できる粒度の短い指示書としてまとめ、1.2featurelist.md の該当節を更新してください。実装そのものは行わないこと。
