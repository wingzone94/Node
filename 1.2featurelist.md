## ① シリーズ（連載）機能 — 完成

### できること

* 記事をシリーズ（`node_series`タクソノミー）にまとめ、記事内に目次・前後ナビ・カード上のバナー（現在回/全話数）を表示
* シリーズ共通のプライマリカラー（term meta）と、記事ごとの色上書き（post meta）の2段階設定。優先順位: 記事上書き > シリーズ共通色 > 既定のブランドオレンジ(`#FF9900`)
* シリーズ内の記事が1件以下の場合は、タクソノミー上の登録は維持したまま記事上には一切表示しない
* シリーズ登録上限10件/シリーズ
* 管理画面「シリーズ内の表示順序」メタボックスで、登録先シリーズ・表示順を両方プルダウンから選択。無効な選択（上限到達済みのシリーズ、既に使用済み/既刊より前の表示順）はUI上で選択不可になる（Ajaxでリアルタイム再計算）
* シリーズtermを削除すると、投稿側に残る表示順メタを自動的に後片付け（記事別カラー上書きは意図的に保持）

## ② ブログカード機能 — 完成（標準埋め込み(oEmbed)のカスタム化、Amazon非対応）

既存機能: `inc/blogcard.php`（OGP取得・キャッシュ・ショートコード・本文中URL単独行の自動カード化）。

> **方針決定: Amazon専用カードは対象外**。`node_render_blogcard()`内の既存`tag=`付与ロジック（Amazonアフィリエイトタグの自動付与）はそのまま維持するが、ASIN抽出・専用UI・ディスクロージャー自動表示など新規のAmazon専用対応は本v1.2では行わない。

### 発端の不具合と方針転換

`core/embed`ブロック（本文中にURLを貼ると自動変換される埋め込み）で、**自サイトの記事URL**は正しくカード化されないが、**他のWordPressサイトの記事URL**（`is-type-wp-embed`）は正しく表示される現象を確認。原因は、現行の`node_auto_blogcard()`が`the_content`フィルタ優先度11で「URL単独行」を正規表現でスキャンする実装のため、それより手前の優先度8で動くWP core標準の`WP_Embed::autoembed()`が先に処理してしまい、正規表現が本文中のURLにマッチしなくなる（=Nodeのブログカード化が素通りされ、WP標準のoEmbed処理結果がそのまま出る）こと。自サイトはWPのREST oEmbedが正常応答しているにも関わらず見た目が崩れる/カード化されないため、**正規表現スキャン方式をやめ、WP標準のoEmbed解決パイプライン自体にフックしてカード化する**方針に転換する。

**スコープ（ユーザー決定・改訂）: oEmbed経由の埋め込みはカード化するが、X(Twitter)とYouTubeは対象外とし、元のオリジナルの埋め込みコード（WP標準のネイティブ埋め込み）をそのまま変更しない**。自サイトの記事・他のWordPressサイトの記事・oEmbedプロバイダのない一般ブログURL等は統一してm3-blogcardデザインにするが、X・YouTubeは現状の挙動に一切手を入れない。

### 新アーキテクチャ（確定事項）

1. **`oembed_dataparse`フィルターでoEmbed成功時を横取り（X/YouTubeは除外）**: `add_filter( 'oembed_dataparse', ... )`で、まず`$data->provider_name`（または`$url`のドメイン）が`Twitter`/`X`/`YouTube`系かどうかを判定する。**該当する場合は`$return`をそのまま返し、一切加工しない**（WP標準のリッチ埋め込みコードを完全に維持）。該当しない場合（自サイトの記事・他のWPサイト・一般ブログ等）のみ、構造化データ（`$data->title` / `$data->thumbnail_url` / `$data->provider_name` / `$data->author_name`）からm3-blogcardを生成して差し替える。判定対象ドメインは`twitter.com` / `x.com` / `youtube.com` / `youtu.be`に固定する。
2. **`embed_maybe_make_link`フィルターでoEmbed失敗時を横取り**: oEmbedプロバイダが存在しない一般的なブログURL等で解決に失敗した場合、WP標準は素のリンクを出すだけになる。この出力を既存の`node_render_blogcard()`（OGPスクレイピング方式）に差し替え、引き続きカード化する（X/YouTubeはoEmbed自体は解決できるため、このフックには通常到達しない）。
3. **`node_auto_blogcard()`（the_content優先度11の正規表現スキャン）は廃止**: 新方式では`WP_Embed::autoembed()`（優先度8、WP core標準）と`do_blocks`（優先度9）が本文中の単独URL・`core/embed`ブロックを先に処理し、上記2フックがその結果に介入するため、優先度11のNode独自スキャンは到達しなくなり不要。`[blogcard url="..."]`ショートコードは引き続き`node_render_blogcard()`を直接呼ぶため維持する。
4. **既存の`node_get_ogp_data()` / `node_render_blogcard()`はフォールバック専用エンジンとして継続利用**。Amazonの`tag=`付与ロジックもこの中に残るため変更不要（Amazonはoembedプロバイダを持たないため自然と`embed_maybe_make_link`経路＝既存ロジックに乗る）。
5. **既存oEmbedキャッシュの無効化が必要**: WPは解決済み埋め込みを投稿メタ`_oembed_<hash>` / `_oembed_<hash>_time`にキャッシュする（既定TTL=`oembed_ttl`フィルタ、デフォルト1日）。既存記事に貼ってある埋め込みは、このキャッシュが切れるまで新デザインに切り替わらない。検証時はキャッシュクリア手段（投稿の再保存、または`wp post meta delete <ID> _oembed_*`相当）を使うこと。

### Phase 1 確定事項（設計完了・GPT-5.5への実装指示）

> **重要な前提発見**: `plugins-embedded/luminous-nexus/luminous-nexus.php` の `register_hooks()` 内に `if ( ! has_filter( 'the_content', 'node_auto_blogcard' ) ) { add_filter( 'the_content', [ $this, 'auto_blogcard' ], 11 ); }` というフォールバックがある。プラグイン側`auto_blogcard()`メソッドは`node_auto_blogcard()`と**全く同じ正規表現バグ**を持つ実装。したがって`node_auto_blogcard()`の`add_filter`登録を単純に外す（または関数を削除する）と、`has_filter`チェックが`false`を返し、プラグイン側の同バグ実装が代わりに有効化されて不具合が再発する。**この理由により、`node_auto_blogcard()`は関数・フィルタ登録ともに削除せず、関数本体だけを無加工パススルー（`return $content;`）に変更し、`add_filter( 'the_content', 'node_auto_blogcard', 11 )`の登録行はそのまま残す**こと（`has_filter`の検出対象として存在させるためのダミー化）。Phase 2aはこの方針に従うこと。

1. **`oembed_dataparse`フック関数シグネチャ**: WP core側の呼び出しは`apply_filters( 'oembed_dataparse', $return, $data, $url )`（`$return`はWP標準が生成したHTML文字列、`$data`はoEmbedレスポンスのstdClass、`$url`は元URL）。よって登録は次の形:
   ```php
   add_filter( 'oembed_dataparse', 'node_oembed_dataparse', 10, 3 );
   function node_oembed_dataparse( string $return, object $data, string $url ): string { ... }
   ```
   X/YouTube判定で除外対象なら**`$return`をそのまま返す**（無加工）。除外対象でなければm3-blogcard HTMLを生成して返す。

2. **X/YouTube除外判定**: ドメインマッチを正とし、`provider_name`文字列マッチを保険として併用する（OR条件、いずれか一致で除外）。
   - ドメイン判定: `parse_url( $url, PHP_URL_HOST )` を小文字化し、`twitter.com` / `x.com` / `youtube.com` / `youtu.be`（およびそれらの`www.`/`m.`サブドメイン）のいずれかに末尾一致するか判定。
   - 保険判定: `$data->provider_name` を小文字化して`twitter`または`youtube`を含むか判定（短縮リンクや転送URLでドメイン判定が外れるケースの保険）。
   - 実装はヘルパー関数`node_is_excluded_oembed_provider( string $url, object $data ): bool`として切り出し、テストから直接呼べるようにすること。

3. **`$data`→m3-blogcard変換ロジック（`$data->type`の扱い）**: `link`/`photo`/`video`/`rich`の型による分岐は行わず、共通化する。
   - 内部/外部判定は`$url`と`home_url()`の文字列比較（`str_contains( $url, home_url() )`）で行う。
   - **内部URLの場合は`$data`を使わず既存`node_get_ogp_data( $url )`に委譲する**（`url_to_postid()`経由で本文・抜粋・サムネイルが取れるため、oEmbedの`title`のみより情報が豊富になり、ショートコード経由の内部カードとも表示が完全に揃う）。
   - **外部URL（oEmbed解決成功）の場合**は`$data`から次のように`$ogp`配列を組み立てる:
     - `title` ← `$data->title ?? ''`（空なら`$url`をフォールバック表示）
     - `description` ← `''`（固定。WP標準oEmbedレスポンスに説明文フィールドは無いため、無理に埋めない。既存テンプレートは`empty($ogp['description'])`で非表示分岐済みなので追加対応不要）
     - `image` ← `$data->thumbnail_url ?? ''`
     - `site_name` ← `$data->provider_name ?? ''`、空なら`parse_url($url, PHP_URL_HOST)`
     - `favicon` ← `'https://www.google.com/s2/favicons?domain=' . rawurlencode( parse_url($url, PHP_URL_HOST) ) . '&sz=64'`（既存`node_get_ogp_data()`の外部ブランチと同じ生成方法に統一）
     - `is_internal` ← `false`
   - 上記いずれの経路で組み立てた`$ogp`配列も、**既存の`node_render_blogcard()`からHTML生成部分を`node_blogcard_markup( array $ogp, string $url ): string`という関数に切り出し、共通利用する**（`node_render_blogcard()`は`node_get_ogp_data()` → `node_blogcard_markup()`の薄いラッパーに変更。Amazon `tag=`付与ロジックは`node_render_blogcard()`側に残したままでよい。oEmbed経由のURLはAmazonがoembedプロバイダから除外済み（`luminous-nexus.php`の`disable_amazon_oembed`）のため、このパスにAmazon URLは到達しない）。

4. **`embed_maybe_make_link`フック**: WP core呼び出しは`apply_filters( 'embed_maybe_make_link', $output, $url )`（2引数）。
   ```php
   add_filter( 'embed_maybe_make_link', 'node_embed_maybe_make_link', 10, 2 );
   function node_embed_maybe_make_link( string $output, string $url ): string {
       $card = node_render_blogcard( $url );
       return $card !== '' ? $card : $output;
   }
   ```
   既存`node_render_blogcard()`にそのまま委譲する（Amazon `tag=`ロジックも内部で維持されるため変更不要）。

5. **`node_auto_blogcard()`の廃止範囲**: 上記の前提発見の通り、**関数定義・`add_filter`登録ともに削除しない**。関数本体を次のように無加工パススルーへ変更する:
   ```php
   function node_auto_blogcard( string $content ): string {
       return $content;
   }
   ```
   `add_filter( 'the_content', 'node_auto_blogcard', 11 );` の行は変更しない。

6. **既存oEmbedキャッシュの無効化手順（Phase 3用）**: WP core標準関数`delete_oembed_caches( int $post_id )`（`wp-includes/media.php`）が`_oembed_*`系postmetaを一括削除する正規の手段。検証時は対象記事IDに対してこれを実行する。
   - WP-CLI: `wp eval 'delete_oembed_caches( <POST_ID> );'`
   - 複数記事を一括クリアしたい場合: `wp post list --post_type=post --field=ID | xargs -I{} wp eval 'delete_oembed_caches( {} );'`
   - SQLで直接確認したい場合（読み取りのみ）: `wp db query "SELECT post_id, meta_key FROM wp_postmeta WHERE meta_key LIKE '_oembed_%' AND post_id = <POST_ID>;"`

   - `node_is_excluded_oembed_provider()`: `https://twitter.com/...`, `https://x.com/...`, `https://www.youtube.com/watch?v=...`, `https://youtu.be/...` → `true`。一般ブログURL・自サイトURL → `false`。

   - `node_embed_maybe_make_link()`: oEmbed失敗時に`node_render_blogcard()`の出力（Amazon `tag=`付与含む）に委譲されること。
   - `node_auto_blogcard()`: 入力contentと出力が完全一致すること（無加工パススルーの確認）、かつ`has_filter( 'the_content', 'node_auto_blogcard' )`が`true`を返すこと（プラグイン側フォールバックが誤って有効化されないことの確認）。

## ③ Node Blocks 埋め込みブロック（node/embed） — 完成

投稿画面（ブロックエディタ）で X(Twitter) / YouTube / Google マップの URL を貼り付けると、`node/embed` カスタムブロックに自動変換され、エディタ内に埋め込みプレビューを表示する。フロント側の出力はテーマの `node_special_embed()`（`inc/blogcard.php`）へ委譲するため、公開ページの HTML は従来と同一。

### 背景

`inc/cleanup.php` で WP 標準の oEmbed REST ルート（`/wp-json/oembed/1.0/proxy`）を無効化しているため、ブロックエディタでは X / YouTube / Google マップの URL を貼り付けても `core/embed` ブロックのプレビューが表示されない（404）。さらに WordPress の `core/embed` twitter variation は x.com の URL を twitter.com に正規化してしまう問題がある。`node/embed` ブロックはこれらの問題を解消する。

### 実装内容

* `plugins-embedded/luminous-blocks/includes/blocks/embed.php` — サーバー側レンダリング（`node_special_embed()` に委譲、エディタ/REST コンテキストでは空を返す）
* `plugins-embedded/luminous-blocks/assets/js/editor.js` — ブロック登録・エディタ UI（ビルド不要のプレーン JS）
  * `wp.data.subscribe` で `core/embed` ブロックの挿入を監視し、対象 URL（X / YouTube / マップ）を検出した瞬間に `node/embed` へ自動置換
  * X: 静的プレースホルダー（X ロゴ・URL・「公開ページでポストが表示されます」メッセージ）。wp-admin コンテキストでは `widgets.js` が正常動作しないため、エディタ内では静的表示にしフロントのみ `widgets.js` でリッチ表示する
  * YouTube: `/embed/` iframe によるプレビュー
  * Google マップ: 16:9 静的プレースホルダー（wp-admin の CSP 制約で iframe 埋め込み不可のため、フロントのみ iframe 出力）
  * PHP 側の `register_block_type` が `render_callback` 付きで先に登録されるため、JS 側で `unregisterBlockType` → `registerBlockType` して edit 関数を確実に使用
* `plugins-embedded/luminous-blocks/includes/oembed-handlers.php` — `node/embed` ブロック登録
* `plugins-embedded/luminous-blocks/luminous-blocks.php` — `embed.php` 読み込み追加、エディタ依存に `wp-i18n` / `wp-api-fetch` 追加
