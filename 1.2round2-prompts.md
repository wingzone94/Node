## ③ 記事総数の正しい取得実装 — 実装（Codex / GPT-5.5）

```
Node テーマ（WordPress、/Users/saitoutatsuya/Documents/GitHub/Node）のv1.2向けに、記事総数取得の共通ヘルパー化と正確化を実装してください。一人で完走すること。

# 背景（現状の問題）
記事総数の取得が箇所ごとにバラバラで、表示される件数が実際の記事一覧・検索対象と一致しないケースがある。現状の呼び出し箇所:
- header.php:462 — 検索モーダルのヒット件数初期値: wp_count_posts()->publish
- template-parts/all-articles.php:7-8 — 記事一覧ページの「公開記事 N 件中」: wp_count_posts('post')->publish
- search.php / inc/search.php — 検索結果件数: $wp_query->found_posts（これは検索クエリ結果なので対象外、触らない）

# 確定仕様
1. inc/utilities.php に共通ヘルパー node_get_total_published_posts(): int を新設する。
   - 基準: 「サイト上で記事一覧・検索の対象になる公開記事の総数」。post_type=post / post_status=publish を正とし、まず現環境で wp_count_posts('post')->publish と実際の一覧件数（例: /all-articles の実表示、検索で全件ヒットさせた際の found_posts）を突き合わせ、差分があるならその原因（除外カテゴリ・特殊メタ等のフィルタ）を特定してヘルパー側で同じ条件を適用すること。差分がなければ wp_count_posts ベースの実装でよい（その場合も「一致を確認した」と報告に明記する）。
   - キャッシュ: transient（キー node_total_published_posts、TTL 12時間）に保存し、save_post / deleted_post / transition_post_status でdelete_transientする無効化フックを併設。
2. header.php:462 と template-parts/all-articles.php の直接呼び出しを node_get_total_published_posts() に置き換える。
3. PHPUnitテスト（tests/配下、既存テスト形式に合わせる）: 公開記事を追加/削除した際にヘルパーの返り値が追随する（transient無効化が効いている）こと、下書き・非公開記事がカウントされないこと。

# 制約（AGENTS.md準拠）
- 変更は inc/utilities.php / header.php / template-parts/all-articles.php / tests/ の4ファイル以内。
- $wp_query->found_posts を使う検索結果表示（search.php, inc/search.php）はスコープ外。触らない。
- php -l で構文チェック。CSSやJSは変更しないので vite build は不要（変更した場合のみ実行）。

# 完了条件
- 検索モーダルの件数と記事一覧ページの件数が同じヘルパー経由になり、実際の公開記事数と一致する
- 記事の公開/下書き戻し/削除で件数表示が追随する（transientが正しく無効化される）
- PHPUnitが全件パスする

# 報告
wp_count_postsと実件数の突き合わせ結果（差分の有無と原因）を必ず報告に含めること。

## ④ プライマリカテゴリの実装 — 実装（Codex / GPT-5.5）

```
Node テーマ（WordPress、/Users/saitoutatsuya/Documents/GitHub/Node）のv1.2向けに、プライマリカテゴリ機能を実装してください。一人で完走すること。

# 機能概要
複数カテゴリ所属記事に「主カテゴリ」を1つ指定でき、パンくず・カテゴリラベルの表示順がそれを優先する。未指定時は現行挙動（先頭カテゴリ）を維持し、後退させない。

# 確定仕様
1. データ: post meta `_node_primary_category`（カテゴリterm_idを文字列で保存）。register_post_meta で single/integer、show_in_rest = true、auth_callback は current_user_can('edit_posts')。
2. 管理UI: 投稿編集画面のサイドに「プライマリカテゴリ」メタボックス（クラシック/ブロックエディタ共通のPHPメタボックスでよい。node-series の「シリーズ内の表示順序」メタボックス実装 plugins-embedded/node-series/node-series.php を参考に、既存の実装パターン・nonce処理に合わせる）。
   - プルダウン: 「指定なし（先頭カテゴリを使用）」+ その記事に割り当て済みのカテゴリ一覧。
   - 保存時バリデーション: 選択term_idがその記事の割り当てカテゴリに含まれない場合はメタを削除（=指定なし扱い）。カテゴリの割り当て解除時（set_object_terms時）も、プライマリが外れていたらメタを削除するクリーンアップフックを追加。
3. 取得ヘルパー: inc/utilities.php に node_get_primary_category( $post_id = null ): ?WP_Term を新設。
   - メタが有効（存在し、かつ記事に割り当て済み）ならそのterm、無効/未指定なら node_get_post_categories_for_display() の先頭を返す。カテゴリなしなら null。
4. 適用箇所:
   - inc/seo.php node_the_breadcrumbs()（seo.php:180-186）: $categories[0] を node_get_primary_category() に置き換え。
   - inc/utilities.php node_get_post_categories_for_display(): プライマリカテゴリを配列の先頭に並べ替えて返す（これにより node_the_category_labels のカード3件制限・シングル表示・article-card.php が自動的にプライマリ優先になる）。
5. PHPUnitテスト（tests/配下）: メタ指定時のヘルパー返り値、未指定時のフォールバック、割り当て外term_id保存時の無効化、カテゴリ解除時のクリーンアップ、node_get_post_categories_for_displayの並び順。

# 制約（AGENTS.md準拠）
- 変更は inc/utilities.php / inc/seo.php / inc/meta-boxes.php（メタボックスは既存のメタボックス集約ファイルがあればそこへ、なければ新規最小ファイル）/ tests/ の5ファイル以内。
- 全体リライト禁止。カテゴリラベルのHTML構造・CSSは変更しない（並び順のみ変わる）。
- php -l で構文チェック。CSS/JS非変更なら vite build 不要。

# 完了条件
- プライマリ指定した記事: パンくずとカテゴリラベル先頭がプライマリカテゴリになる
- 未指定の記事: 表示が現行と完全に同一（後退なし）
- 割り当てから外したカテゴリがプライマリに残留しない
- PHPUnitが全件パスする

# 報告
変更ファイル、テスト結果、cybernode.localで確認すべき記事パターン（複数カテゴリ+指定あり/なし）を報告すること。
```
