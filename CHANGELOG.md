## [1.2.5] - 2026.07.24

### 読了時間・文字数バッジを常時1行表示に（表示内容は1.2.4のまま）
- **バッジの表示内容は 1.2.4 のまま**（「約◯分で読めます」＋文字数チップ。チップは長さ判定に応じた配色＝短い緑／標準青／長い赤 で従来どおり着色）。ランク語（「やや長い」等）の文字追加や本文行への文字数移設は**行わない**。
- **狭い右カラム（PC ~280px）やモバイル展開時に折り返していたのを `nowrap` で常に1行に収める**よう `_hero-toc.css` を調整。この文脈のみ字送りを詰め、最長ケース（約24分・約13,000文字）でも見切れず1行に収まることを実測確認。
- **フォントは Inter / Noto Sans JP（weight 400）を維持**。開発中の Node 1.3 のフォント移行（Hiragino Sans W6 追加）は含めず、記事本文・メンテナンス画面ともに重い Hiragino W6 は使用しない（メンテナンス画面はシステムフォントの独立スコープ）。本リリースは 1.2.4 を基点とし、他の Node 1.3 項目も含まない。
- **メンテナンスモードを追加**: 設定 → Luminous Settings から有効化すると、サイトのフロント側に専用のメンテナンス画面（HTTP 503 + `Retry-After`）を表示します。**管理画面とログイン画面は影響を受けない**ため、有効中でも解除できます。
- **復旧予定時刻とカウントダウン**: 復旧予定時刻を設定すると、メンテナンス画面に残り時間のカウントダウンと、開始から復旧予定までの**進捗ゲージ**を表示します（未設定なら非表示）。端末の時計のずれはサーバー時刻で補正します。
- **管理者にも同じ画面を表示**し、画面内に管理画面へ戻るリンクを出します（訪問者に見えている状態をそのまま確認できるようにするため）。管理バーと管理画面上部にも「メンテナンス中」を常時表示し、解除忘れを防ぎます。
- **開始・終了を Discord へ通知**（node-connect v1.3.6）: 予約済みだった `maintenance_start` / `maintenance_end` イベントを実際に発火させ、購読設定の対象に追加しました。復旧予定時刻も通知に含まれます。作業中の入切を握り潰さないよう、この2イベントは重複抑止の対象外にしています。
- メンテナンス画面はテーマのヘッダー・フッターやビルド済みCSSに依存せず1ファイルで完結させています（不調が起きている場面で使う画面のため、依存を増やして画面自体が壊れることを避ける意図）。ダークモード・モバイル幅に対応。

## [1.2.3] - 2026.07.20

### Node Connect をテーマ同梱プラグインに変更（node-connect v1.3.5）
- **node-connect をテーマの同梱プラグイン一覧（`$embedded_plugins`）に追加**: これまで node-connect だけが同梱対象から漏れており、更新のたびに `production_plugins/node-connect.zip` を管理画面から手動インストールする必要があった。**テーマ更新だけで node-connect も更新される**ようになり、他の9プラグインと運用が揃う。
- node-connect に初期化関数 `node_connect_init()` を追加（他の同梱プラグインと同形式）。
- ※ 既に単体プラグインとして「Node Connect」をインストール済みの環境では、そのプラグインを**停止・削除**すると同梱版に切り替わります（設定・送信履歴はDBに残るため引き継がれます）。

#### 同日修正: 単体版 Node Connect 導入環境でサイト全体が停止する不具合（同一バージョンでzip更新）
- **初回配信の 1.2.3 には、単体プラグインとして Node Connect を導入済みの環境でサイト全体が致命的エラーになる不具合があった**（`Cannot redeclare class Node_Connect`。フロント・管理画面とも表示不能）。テーマ側の同梱ローダーは初期化関数 `node_connect_init()` の有無で二重読み込みを判定していたが、**1.3.3 以前の単体プラグインはこの関数を持たない**ため判定をすり抜け、テーマ同梱版が二重に読み込まれていた。
- 対策を二重化: ①テーマ側ローダーが、同名プラグインが**単体で有効化されている場合は同梱版を読み込まない**よう `active_plugins` を参照（全同梱プラグイン共通の防御）。②node-connect 側を**冪等化**し、既に読み込み済みなら何もしないようにした。
- 冪等化の判定には `class_exists()` を使えない（PHPは無条件のクラス宣言をコンパイル時に巻き上げるため、自分自身の宣言を検知して定数未定義のまま return してしまう）。実行時にのみ定義される定数で判定し、あわせて class / function を条件付き宣言にして巻き上げ自体を回避している。
- 復旧方法（初回配信版で障害が出た場合）: FTP等で `wp-content/plugins/node-connect/` を削除すると同梱版に切り替わり復旧します。

## [1.2.2] - 2026.07.20

### Node Connect: 通知設定のトグルUI化（node-connect v1.3.3・別配布プラグイン）
- **外部連携設定のチェックボックスをすべてトグルスイッチに変更**: 全体の有効化・一時停止・Webhookごとの有効化・イベント購読・X自動投稿の各設定を、状態が一目で分かるトグル（オン=ブランドオレンジ / オフ=グレー）に統一。
- **「記事の非公開化」など個別イベントの通知をトグルで停止可能に**: 従来もチェックボックスで購読を外せば止まったが、状態が分かりにくかったため視認性を改善。
- **親設定がオフのときは配下をグレーアウトして操作不可に**: 「Webhook通知」をオフにすると全設定が、個別Webhookをオフにするとそのイベント購読がグレーアウトされる。グレーアウト中も `disabled` は使わず値を保持して送信するため、**その状態で保存しても購読設定やWebhook URLが消えない**。
- ※ node-connect はテーマに同梱されず、`production_plugins/node-connect.zip` を WordPress 管理画面からインストールして更新します。

### シリーズカードの見出しをシリーズ名表示に変更
- **記事下シリーズカードの折りたたみ時ラベル「目次」をシリーズ名そのものの表示に変更**: 本文の見出しジャンプ用「目次」（ヒーロープルダウン・サイドバー・モバイルFAB）と紛らわしかったため。「N / M」の現在回バッジは維持。
- **シリーズ名の自動フィット**: 名前がカード幅に収まらない場合、字詰め（letter-spacing 最大-0.05em）→縮小（font-size 最小0.85rem）の順に自動調整し、それでも溢れる場合は末尾を省略記号で切り詰める。Webフォント適用後・リサイズ時にも再フィット。
- アクセシビリティ: トグルの `aria-label` を「シリーズ「◯◯」の記事一覧」に変更。

## [1.2.1] - 2026.07.19

### シリーズ（連載）機能のホットフィックス（同日zip更新）
- **第1回のみ（1件）のシリーズでも目次・カードバナーを表示するように変更**: 従来は「シリーズ内の公開記事が1件以下は記事上に一切表示しない」仕様だったため、連載の第1回を公開・執筆した時点ではシリーズ設定がどこにも反映されず「設定が効いていない」ように見えていた（node-series v1.2.1）。
- **未公開記事のプレビューで自分自身をシリーズ一覧に含めるように修正**: 目次・位置情報（N/全M回）が公開記事のみを集計していたため、下書き・レビュー待ちのプレビューでは自記事が数えられず表示確認ができなかった。合流させるのは表示中の記事自身のみで、他の未公開記事が目次に漏れることはない。

### node-connect ベータ版を同梱（Node 1.3 Connect 先行検証）
- **新規プラグイン `node-connect`（v1.3.0-beta.1）をテーマに同梱**: 記事の新規公開（予約公開を区別）・公開済み記事の更新・非公開化・削除の各イベントを Discord Incoming Webhook（Embed形式）へ自動通知する Webhook 基盤。設定 → 外部連携から、Webhook URL（最大3件・URL単位でイベント購読/停止を選択）・全体有効化・一時停止・接続テスト・送信履歴（直近50件）を管理。
- 安全設計: 送信は `wp_schedule_single_event` による非同期化で記事公開処理に波及させない。タイムアウト5秒・失敗時は最大2回再送（計3試行）。`post_id + イベント + URL` の transient（10分）による重複送信防止。Webhook URL はフロントへ一切出力せず、設定画面でも末尾数文字以外をマスク表示。
- Embed カラーはシリーズのプライマリカラー → 無ければブランドオレンジ `#FF9900`。削除・非公開の通知にはリンクを付けない。
- AI要約完了・ファクトチェック完了・AI処理失敗のイベント名を予約（発火は Node 1.3 本編で追加予定）。
- テーマ側: テーマ更新のインストール成功時に `node_connect_event`（Nodeアップデート）を発火。
- インストール用 ZIP `production_plugins/node-connect.zip` を追加（WP管理画面からアップロード可能なトップレベル `node-connect/` 構造）。

## [1.2.0] - 2026.07.18

### シリーズ（連載）機能
- **記事をシリーズにまとめる機能を新設**: `node_series` タクソノミーで記事を連載としてまとめ、記事内に目次・前後ナビ・カード上のバナー（現在回/全話数）を表示。シリーズ共通のプライマリカラー（term meta）と記事ごとの色上書き（post meta）の2段階設定に対応（優先順位: 記事上書き > シリーズ共通色 > ブランドオレンジ `#FF9900`）。登録上限は10件/シリーズ。シリーズ内の記事が1件以下の場合は記事上に一切表示しない。
- 管理画面「シリーズ内の表示順序」メタボックスで登録先シリーズ・表示順をプルダウン選択。無効な選択（上限到達済み・使用済みの表示順）はAjaxのリアルタイム再計算でUI上選択不可に。ブロックエディタ／クラシックエディタの双方で動作。
- シリーズ term の削除時に投稿側の表示順メタを自動で後片付け（記事別カラー上書きは意図的に保持）。

### ブログカード / 埋め込み
- **ブログカードを oEmbed パイプライン方式へ刷新**: 自サイト記事URLがカード化されない不具合を修正。`the_content` 優先度11の正規表現スキャン（`node_auto_blogcard()`）を廃止し、`oembed_dataparse`（解決成功時）と `embed_maybe_make_link`（解決失敗時）にフックする方式へ変更。自サイト記事・他のWordPressサイト・一般ブログURLを統一して `m3-blogcard` デザインで表示。
- **X(Twitter) と YouTube は対象外**とし、WP標準のネイティブ埋め込みをそのまま維持。
- **埋め込みブロック `node/embed` を新設**: エディタで X / YouTube / Google マップのURLを貼ると `node/embed` に自動変換し、エディタ内にプレビューを表示。フロント出力は従来と同一（`node_special_embed()` へ委譲）。
- 既存記事の埋め込みは投稿メタ `_oembed_*` のキャッシュが切れるまで旧デザインのまま表示されます（再保存で即時反映）。

### 記事ヒーロー
- **PCヒーローを縦2カラム構成へ再設計**: 左に核心（カテゴリ→タイトル→メタ）、右に補足（読了→著者→開示バッジ→目次）。1000px以下では従来同様の1カラムへ自動で折りたたみ。
- **目次をヒーローへ統合**（ネイティブ `<select>` ベースのプルダウン、マルチページ記事はページ単位でグループ化）。モバイルはヒーロー通過後に目次FAB＋パネルを解放（`body.is-hero-passed`）。
- **読了バッジを簡素化**: 「約◯分で読めます」＋「約N,NNN文字」チップに変更。読了時間は**550字/分の固定換算**、長さ判定は**絶対文字数**（〜1,500短い / 〜3,000やや短い / 〜6,000標準 / 〜10,000やや長い / 以上長い）に改定（旧「サイト平均比」の相対換算による異常値を解消）。ランク連動の配色（短い=緑 / 標準=青 / やや長い=橙 / 長い=赤）は 1.1.x と同様に維持（一時的にブランドオレンジ固定へ変更したが 7/18 に復元）。
- **AI開示チップを静音化**: オレンジ塗りのCTA風からアウトラインの情報チップへ。アイコン色を可読色に固定（動的カラーで灰青に化ける不具合の修正）。
- アイキャッチを幅基準の縮尺で全体表示に統一（16:9固定枠・レターボックスを廃止）。画像なしヒーローは上詰めに。
- 投稿日を日付アーカイブへのリンクに（追記日は表示のみ）。コメントは1件以上のときだけカウンターを表示。

### 脚注 / スクロール（2026-07-18 追補）
- 脚注の「↩（戻る）」リンクで戻った位置が固定ヘッダーに隠れる不具合を修正（本文中の脚注参照アンカーに `scroll-margin-top` を追加。脚注へ飛ぶ方向は従来から対応済み）。
- 脚注のページ分割タブがある記事で、ページ読み込み直後に脚注セクションまで勝手にスクロールする不具合を修正（タブ初期化時の `scrollIntoView` がページ全体を縦スクロールさせていた。初期化時はタブ帯内の横位置調整のみ行う）。

### ヘッダー（2026-07-18 追補）
- **ログイン中に WordPress 管理バーとテーマヘッダーが衝突する不具合を修正**: スクロール時のヘッダー退避量が管理バーのオフセット（PC 32px / モバイル 46px）を考慮しておらず、隠れたヘッダーが画面上部に残留していた。特にモバイル幅（600px以下）では管理バーがスクロールで画面外へ去るため、46px分のヘッダー帯が本文に重なり続けていた。退避量を `calc(-100% - オフセット)` に修正し、モバイル幅ではヘッダーの `top` を管理バーの見えている下端に追従させて隙間・重なりを解消。非ログイン時の挙動は不変。

### シェア
- システムシェア（OSのシェアシート）ボタンが iOS / Android / Mac / Windows で表示されない不具合を修正。`wp_is_mobile()` によるサーバー側UA判定はキャッシュ済みHTMLとUAが食い違うため廃止し、常に `hidden` で出力して Web Share API（`navigator.share`）対応ブラウザのみクライアント側で表示する方式に変更。※ Web Share API はHTTPS必須のため、`http://` のローカル環境ではボタンは表示されません。

### 記事カード
- **カード全面クリック**（stretched link 方式）。カテゴリピル等は個別操作を維持。
- グリッドの行内高さを統一（`align-items: stretch`）。
- **追記日を表示**（同年なら「追記 7/11」の短縮表記で投稿日と1行表示）。判定は `node_get_post_modified_display()` に共通化。
- AI・スポンサーの開示バッジをアイキャッチ上のオーバーレイからカテゴリ行右端へ移動（衝突解消）。
- カード上部（カテゴリ行の上）に不要な境界線が表示される不具合を修正（フッター用 `border-top` のセレクタ書き残しが原因）。

### Node Library
- 記事の動的カラーで `--md-sys-color-primary` が灰色に解決される記事（例: グレー基調のアイキャッチ）で、ゲームカードの「詳細を見る」とヒーローのNode Libraryピルのアイコンが灰色化して読めなくなる不具合を修正。「詳細を見る」は黒系（on-surface）固定、ピルのアイコン・ホバー色はAI開示チップと同じブランドオレンジ固定に。本文埋め込み時に「詳細を見る」へ本文リンク様式（青字・下線）が上書きされる問題も除外リスト追加で解消。
- **ライブラリを新設計**（ストアフロント一覧＋個別ページ）。タイプフィルターはプリティURL（`/node-library/game/` 等）でのサーバーサイド絞り込み。「この作品に触れた記事」の逆引きにも対応。
- 記事ヒーローからライブラリへのクイックリンクを追加。

### アーカイブ / ナビゲーション
- **日付アーカイブを新設計**（`date.php`）: 日別グループのタイムライン型リスト＋期間セレクタ＋前後ナビ。
- **404ページを新設**（`404.php`）: 検索フォームとホームへの導線を備えた軽量デザイン。noindex / nofollow を付与。

- モバイルのボトムバーを廃止し、トップFABのみに整理。

### 修正 / 内部
